Criando CI/CD do blog com Wyam, Azure Devops e Github Pages

Published on Friday, April 10, 2020
azuredevops githubpages wyam

** DRAFT **

Este post não foi tão rapido de escrever mas a execução dele deve ser a partir de agora.

Eu vinha mantendo os inputs do Blog na minha maquina e sem versionamento, mas comecei a usar um outro notebook e pensar em copiar/colar o conteúdo prebuild do blog pra conseguir postar conteúdo era a motivação que eu precisava para fazer o CI/CD.

Neste post vamos abordar a criação de um CI/CD, commitando os inputs do blog em um repositório, criando a trigger de commit no master para popular o conteúdo em outro repositório também do Github. É uma implementação que não segue muitos padrões/não muito convencional, já que nosso ponto de publicação é um repositório do git.

Passos:

  • criar chave de deployment no projeto
  • criar pipeline no azure devops
  • adicionar a chave em secure files na library do azure devops
  • conectar pipeline no repositório de inputs
  • criar script de build
  • dar permissão no secure file após commit na origem
